# Heading Hierarchy Guide — almadigitaldesigns.com
## For: Apollo (WD)

### Why This Matters
- Google uses H1-H3 to understand page structure and topic hierarchy
- Only ONE H1 per page — it tells Google what the page is "about"
- H2s are major sections; H3s are sub-points within those sections
- Never skip levels (e.g., H1 -> H3 without an H2)
- Headings must contain target keywords naturally — do not keyword-stuff

### H4, H5, H6
- **H4:** Use for sub-items within H3 sections (e.g., feature lists within a pricing tier)
- **H5/H6:** Avoid unless genuinely needed — over-use dilutes SEO value
- Footer nav categories can use H4 or H5

## Common Mistakes to Avoid
- DO NOT use headings purely for visual styling — use CSS classes instead
- DO NOT put the logo or company name in the H1 (that is what the `<title>` tag is for)
- DO NOT use H2 for the footer copyright line
- DO NOT duplicate the H1 text in any H2
- DO wrap section labels (e.g., "PRICING", "PORTFOLIO") in a `<span>` or `<p>` with a styling class if they are purely decorative labels, not semantic headings
